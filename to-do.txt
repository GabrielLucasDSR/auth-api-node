# Auth API - Technical Roadmap

## Done

- [x] Layered architecture (routes -> controllers -> services -> repositories).
- [x] JWT auth flow with access token + refresh token.
- [x] Refresh token rotation with unique jti.
- [x] Refresh token revocation (logout).
- [x] Refresh token hash-at-rest (tokenHash) persisted in database.
- [x] JWT_SECRET fail-fast validation at startup.
- [x] Zod payload validation for auth routes.
- [x] Centralized error handling with AppError pattern.
- [x] Structured logging with correlation id.
- [x] Health/readiness endpoints.
- [x] Swagger/OpenAPI docs endpoint.
- [x] Redis-backed rate limiting with safe in-memory fallback.
- [x] Session management endpoints:
- [x] GET /auth/sessions
- [x] POST /auth/logout-session
- [x] POST /auth/logout-all
- [x] Unit + middleware + repository + e2e test suites.
- [x] CI pipeline running lint + tests + coverage threshold.

## In Progress

- [ ] Investigate and eliminate intermittent Jest open handles warning.

## Next Priorities

- [ ] Improve branch coverage on middleware/controller negative paths.
